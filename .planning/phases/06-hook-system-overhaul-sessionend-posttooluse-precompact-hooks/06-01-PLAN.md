```go
// TranscriptResult holds parsed JSONL transcript data.
type TranscriptResult struct {
    Tokens          map[string]TokenBreakdown // per-model token breakdown
    CompactionCount int
    LastModel       string // last seen model ID
    ProjectPath     string // from cwd field
}

// ParseTranscript reads a JSONL transcript file and extracts per-model token
// breakdowns and compaction count. This replaces the inline parseJSONLSession
// from main.go with a reusable function for hook-triggered reads (D-01, D-02).
// Uses 1MB scanner buffer matching the original implementation.
func ParseTranscript(path string) (TranscriptResult, error)
```

Implementation mirrors main.go parseJSONLSession logic (lines 723-796):
- Open file, create bufio.Scanner with 1MB buffer
- Parse each line as JSON: type, timestamp, cwd, isCompactSummary, message.model, message.usage.{input_tokens, output_tokens, cache_read_input_tokens, cache_creation_input_tokens}
- Accumulate per-model TokenBreakdown (Input, Output, CacheRead, CacheWrite)
- Count isCompactSummary as compactions
- Track lastModel and projectPath
- Return TranscriptResult

Per D-02: Internal throttling (max 1 read per 10s) is NOT in ParseTranscript itself — throttling is handled by the caller (server hook handler). ParseTranscript is a pure read function.
