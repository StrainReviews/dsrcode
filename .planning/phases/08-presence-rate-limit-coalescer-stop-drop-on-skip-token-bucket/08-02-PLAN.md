    ```go
    package coalescer

    import (
        "hash/fnv"

        "github.com/StrainReviews/dsrcode/discord"
    )

    // sep is the ASCII Unit Separator (0x1F). It is reserved in the ASCII
    // control-code block specifically for delimiting fields — it effectively
    // never appears in user content. See 08-RESEARCH.md §Pattern 4 +
    // discrepancy D3 (supersedes 08-CONTEXT.md D-12 which originally
    // specified \x00; 0x1F is the correct choice).
    const sep byte = 0x1F

    // HashActivity produces a stable 64-bit FNV-1a hash of the user-visible
    // fields of an Activity. Used by the Coalescer to skip Discord
    // SetActivity calls when the content has not changed since the last
    // successful push.
    //
    // StartTime is intentionally EXCLUDED from the hash (08-CONTEXT.md D-09):
    // it is deterministic per-session (the earliest active session's
    // StartedAt, verified via resolver/resolver_test.go:333-339), but the
    // pointer identity and monotonic-wall offset can produce different byte
    // representations for the same logical instant. Including it would
    // trigger a spurious hash difference on every resolve cycle.
    //
    // FNV-1a is NOT cryptographic. It is a content-identity hash only — a
    // 64-bit collision is possible but vanishingly rare (~2^32 birthday
    // bound) and the worst outcome is one legitimate update skipped; the
    // next distinct resolve auto-recovers.
    func HashActivity(a *discord.Activity) uint64 {
        if a == nil {
            return 0
        }
        h := fnv.New64a()
        h.Write([]byte(a.Details))
        h.Write([]byte{sep})
        h.Write([]byte(a.State))
        h.Write([]byte{sep})
        h.Write([]byte(a.LargeImage))
        h.Write([]byte{sep})
        h.Write([]byte(a.LargeText))
        h.Write([]byte{sep})
        h.Write([]byte(a.SmallImage))
        h.Write([]byte{sep})
        h.Write([]byte(a.SmallText))
        h.Write([]byte{sep})
        // Buttons preserve insertion order (slice), so iteration is
        // deterministic by construction.
        for _, b := range a.Buttons {
            h.Write([]byte(b.Label))
            h.Write([]byte{sep})
            h.Write([]byte(b.URL))
            h.Write([]byte{sep})
        }
        return h.Sum64()
    }
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Resolver output → HashActivity | discord.Activity values originating from resolver.ResolvePresence (in-process, trusted) |
| HashActivity → lastSentHash | uint64 value placed into atomic slot, read-only from Coalescer's flush path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | Tampering | Adversarial Activity input crafted to collide with lastSentHash | accept | Resolver is fully in-process; no external input reaches HashActivity. FNV-1a 64-bit collision for any two legitimate Activities has ~2^32 birthday probability. Worst case: one update skipped; next distinct update auto-recovers. |
| T-08-02-02 | Information Disclosure | Hash value leaks sensitive session content via logs | mitigate | `slog.Debug("presence update skipped", "reason", "content_hash")` emits NO hash value in the log. The hash is a uint64 stored in memory only. |
| T-08-02-03 | Denial of Service | Resolver returning frequently-changing-but-equivalent Activity defeating the skip | mitigate | StartTime exclusion (D-09) is the primary mitigation. Field list is frozen; if a future resolver adds a new field, the plan author must consciously decide whether to include it in HashActivity. |
| T-08-02-04 | Tampering | `json.Marshal` map-ordering shifts between Go versions → hash drift | mitigate | Hashing uses field-ordered raw-bytes (D-11). No `json.Marshal`, no `fmt.Sprintf`. Grep-enforceable. |
| T-08-02-05 | Denial of Service | IPC failure → lastSentHash poisoned with un-delivered content → next retry wrongly skipped | mitigate | Hash is stored ONLY after a successful SetActivity (not before, not on error). Failed flushes leave lastSentHash unchanged. |
| T-08-02-06 | Elevation of Privilege | n/a | — | No new access surface. |
