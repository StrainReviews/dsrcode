```go
package server

import (
    "context"
    "encoding/json"
    "fmt"
    "log/slog"
    "net"
    "net/http"
    "os"
    "path/filepath"
    "strconv"
    "time"

    "github.com/tsanva/cc-discord-presence/session"
)

// ActivityMapping maps tool names to (SmallImageKey, SmallText) per D-07.
var ActivityMapping = map[string]struct{ Icon, Text string }{
    "Edit":      {"coding", "Editing a file"},
    "Write":     {"coding", "Writing a file"},
    "Bash":      {"terminal", "Running a command"},
    "Read":      {"reading", "Reading a file"},
    "Grep":      {"searching", "Searching codebase"},
    "Glob":      {"searching", "Searching files"},
    "WebSearch": {"searching", "Searching the web"},
    "WebFetch":  {"searching", "Fetching web content"},
    "Task":      {"thinking", "Running a subtask"},
}

// HookPayload is the JSON body from Claude Code HTTP hooks.
type HookPayload struct {
    SessionID string `json:"session_id"`
    Cwd       string `json:"cwd"`
    ToolName  string `json:"tool_name,omitempty"`
}

// StatuslinePayload matches Claude Code's statusline JSON structure.
type StatuslinePayload struct {
    SessionID string `json:"session_id"`
    Cwd       string `json:"cwd"`
    Model     struct {
        ID   string `json:"id"`
        Name string `json:"name"`
    } `json:"model"`
    TotalTokens  int64   `json:"total_tokens"`
    TotalCostUSD float64 `json:"total_cost_usd"`
}

// ConfigUpdatePayload for POST /config endpoint per D-50.
type ConfigUpdatePayload struct {
    Preset string `json:"preset,omitempty"`
}

// Server manages the HTTP server for hook events.
type Server struct {
    registry  *session.SessionRegistry
    onConfig  func(ConfigUpdatePayload)
    httpServer *http.Server
}

// NewServer creates a new Server.
// registry: session registry for activity updates
// onConfig: callback for config change requests
func NewServer(registry *session.SessionRegistry, onConfig func(ConfigUpdatePayload)) *Server {
    return &Server{
        registry: registry,
        onConfig: onConfig,
    }
}
```

Implement route handlers:

**handleHook(w, r)** — `POST /hooks/{hookType}`:
- Decode JSON body as HookPayload
- Get hookType from `r.PathValue("hookType")`
- Map hookType to activity:
  - "pre-tool-use": look up ToolName in ActivityMapping. If not found, use ("thinking", "Processing...")
  - "user-prompt-submit": ("thinking", "Thinking...")
  - "stop": ("idle", "Finished")
  - "notification": ("idle", "Waiting for input")
- Build ActivityRequest with mapped icon/text
- Get PID from header or default to 0 (Claude Code may not send PID, use os.Getppid() as approximation)
- If session doesn't exist for this SessionID, call registry.StartSession first
- Then call registry.UpdateActivity
- Return 200 empty body (fastest response per hook docs)
- Log at debug level: "hook received", "hookType", hookType, "tool", payload.ToolName

**handleStatusline(w, r)** — `POST /statusline`:
- Decode JSON body as StatuslinePayload per D-36/D-37
- Get git branch from cwd using `exec.Command("git", "-C", cwd, "rev-parse", "--abbrev-ref", "HEAD")`
- Call registry.UpdateSessionData with model name, branch, tokens, cost
- Return 200 empty body

**handleConfigUpdate(w, r)** — `POST /config` per D-50:
- Decode ConfigUpdatePayload
- If Preset is non-empty, call onConfig callback
- Return 200 with `{"ok": true}`

**handleHealth(w, r)** — `GET /health`:
- Return 200 with JSON: `{"status": "ok", "sessions": registry.SessionCount(), "uptime": time.Since(startTime).String()}`

**handleSessions(w, r)** — `GET /sessions`:
- Return 200 with JSON array of all sessions from registry.GetAllSessions()

**Start(ctx context.Context, addr string, port int) error**:
- Create ServeMux, register all routes
- Create http.Server with Addr = net.JoinHostPort(addr, strconv.Itoa(port))
- Start in goroutine
- Select on ctx.Done() for graceful shutdown with 5s timeout
- Per D-55: addr defaults to "127.0.0.1"

**MapHookToActivity(hookType string, toolName string) (icon string, text string)**:
- Export this function so tests can verify the mapping directly
- Implement the mapping logic described above
