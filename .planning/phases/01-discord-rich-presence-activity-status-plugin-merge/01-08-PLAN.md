Rewrite main.go. Keep the existing JSONL fallback functions but restructure the main flow.

**CLI flags** (per D-33, D-53):
```go
var (
    flagPort    = flag.Int("port", 0, "HTTP server port (default 19460)")
    flagPreset  = flag.String("preset", "", "Display preset name")
    flagVerbose = flag.Bool("v", false, "Debug logging")
    flagQuiet   = flag.Bool("q", false, "Error-only logging")
    flagConfig  = flag.String("config", "", "Config file path")
)
```

**main() flow:**

1. Parse flags
2. Load config: `cfg := config.LoadConfig(*flagPort, *flagPreset, *flagVerbose, *flagQuiet, *flagConfig)`
3. Setup logger: `logger.Setup(cfg.LogFile, cfg.LogLevel)`
4. Log startup: `slog.Info("starting cc-discord-presence", "version", "2.0.0", "port", cfg.Port, "preset", cfg.Preset)`
5. Load preset: `currentPreset := preset.MustLoadPreset(cfg.Preset)` — store in a `sync.RWMutex`-protected variable for hot-reload
6. Create context with cancel for graceful shutdown: `ctx, cancel := context.WithCancel(context.Background())`
7. Signal handling: `signal.Notify(sigChan, syscall.SIGINT, syscall.SIGTERM)` — on signal, call cancel()

8. Create Discord client: `discordClient := discord.NewClient(cfg.DiscordClientID)` — use the configured client ID. If empty, use the existing `ClientID` constant as fallback. Per D-16, the new app "DSR Code" will have its own ID.

9. **Discord connection loop** (goroutine):
   - Use `discord.DefaultBackoff()` for reconnection per D-42
   - Try Connect(). On success, Reset() backoff, log info. On failure, log error, sleep Next() duration.
   - Retry until ctx cancelled
   - Per D-43: silent failure on activity updates — log at debug level only

10. **Presence update debouncer** (goroutine):
    - Channel-based: registry.onChange sends signal to a channel
    - Debounce 100ms after last signal, then:
      - Read all sessions from registry
      - Call resolver.ResolvePresence(sessions, currentPreset, time.Now())
      - If activity != nil, call discordClient.SetActivity(activity)
      - If activity == nil, clear presence
    - Rate limit: track last update time, skip if < 15s since last SetActivity (Discord rate limit)
    - Per D-43: on SetActivity error, log debug, attempt reconnect

11. Create session registry with onChange callback that sends to the debounce channel:
    ```go
    updateChan := make(chan struct{}, 1)
    registry := session.NewRegistry(func() {
        select {
        case updateChan <- struct{}{}:
        default: // non-blocking
        }
    })
    ```

12. Start HTTP server: `srv := server.NewServer(registry, onConfigUpdate); go srv.Start(ctx, cfg.BindAddr, cfg.Port)`
    - onConfigUpdate reloads the preset when POST /config is called

13. Start stale checker: `go session.CheckStaleSessions(ctx, registry, cfg.IdleTimeout, cfg.RemoveTimeout, cfg.StaleCheckInterval)`

14. Start config watcher: `go config.WatchConfig(ctx, configPath, func(newCfg config.Config) { ... })` 
    - On config change: reload preset, update log level

15. **JSONL fallback** (per D-51): Keep the existing fsnotify watcher for JSONL files. When JSONL data arrives AND no HTTP session exists for that project, create a "passive" session from JSONL data. This provides the "zero-config install and forget" experience. The existing `readSessionData()` and `readJSONLData()` functions should feed into the registry via `registry.UpdateSessionData()`.
    - Preserve the existing file-watching goroutine but adapt it to use the registry instead of directly updating Discord

16. Wait for ctx.Done() (signal received), log "shutting down", exit cleanly.
