**Create `.goreleaser.yaml`** at repo root (`C:/Users/ktown/Projects/cc-discord-presence/.goreleaser.yaml`) with this exact content:

```yaml
# yaml-language-server: $schema=https://goreleaser.com/static/schema.json
version: 2

project_name: cc-discord-presence

builds:
  - id: cc-discord-presence
    main: .
    binary: cc-discord-presence
    env:
      - CGO_ENABLED=0
    goos:
      - darwin
      - linux
      - windows
    goarch:
      - amd64
      - arm64
    ignore:
      - goos: windows
        goarch: arm64
    ldflags:
      - -s -w -X main.version={{.Version}}
    mod_timestamp: "{{ .CommitTimestamp }}"

archives:
  - id: default
    formats:
      - tar.gz
    format_overrides:
      - goos: windows
        formats:
          - zip
    name_template: >-
      {{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}
    files:
      - LICENSE
      - README.md
      - CHANGELOG.md
      - PRIVACY.md
      - TERMS.md

checksum:
  name_template: "{{ .ProjectName }}_{{ .Version }}_checksums.txt"
  algorithm: sha256

changelog:
  use: github-native

release:
  github:
    owner: StrainReviews
    name: dsrcode
  prerelease: auto
  make_latest: true
```

Key design points:
- `version: 2` (GoReleaser v2 format, NOT v1)
- `formats: [tar.gz]` (v2 list syntax, NOT `format: tar.gz`)
- `format_overrides.formats: [zip]` for Windows (v2 list syntax)
- `ignore: windows/arm64` produces exactly 5 targets per DIST-03
- `ldflags: -s -w -X main.version={{.Version}}` strips debug info and injects version
- `release.github.owner: StrainReviews`, `name: dsrcode` per DIST-10
- `checksum.algorithm: sha256` per DIST-05

**Replace `.github/workflows/release.yml`** with this exact content:

```yaml
name: Release

on:
  push:
    tags:
      - "v*"
  workflow_dispatch:
    inputs:
      tag:
        description: "Tag to release (e.g., v3.2.0)"
        required: true
        type: string

permissions:
  contents: write

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Create tag (workflow_dispatch only)
        if: github.event_name == 'workflow_dispatch'
        run: |
          git tag "${{ inputs.tag }}"

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version-file: go.mod

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v7
        with:
          distribution: goreleaser
          version: "~> v2"
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Key design points per DIST-02:
- `on.push.tags: ["v*"]` triggers on tag push
- `on.workflow_dispatch` with tag input enables GitHub UI Release button
- `workflow_dispatch` creates the tag locally before goreleaser runs
- `fetch-depth: 0` required for changelog and version detection
- `go-version-file: go.mod` reads Go version from source of truth (fixes current hardcoded `1.26` bug)
- `goreleaser-action@v7` (latest, matches v2 defaults -- CONTEXT says v6 but research shows v7 is current and @v6 is outdated; using v7 per research recommendation)
- `version: "~> v2"` pins to GoReleaser v2.x semver range
