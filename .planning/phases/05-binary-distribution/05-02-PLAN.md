2. Platform detection function `detect_platform()`:
   - `uname -s` with case mapping: `mingw*|msys*|cygwin*` -> `windows`, `darwin` -> `darwin`, `linux` -> `linux`
   - `uname -m` with case mapping: `x86_64|amd64` -> `amd64`, `aarch64|arm64` -> `arm64`
   - Set `IS_WINDOWS=true/false`, `PLATFORM_OS`, `PLATFORM_ARCH`

   - `BINARY_NAME="cc-discord-presence"` (simple name, no OS-arch suffix -- inside PLUGIN_DATA we know the platform)
   - If Windows: `BINARY_NAME="cc-discord-presence.exe"`

4. Migration function `migrate_old_binary()` (runs once):
   - Check for binary at old location `$HOME/.claude/bin/cc-discord-presence-${PLATFORM_OS}-${PLATFORM_ARCH}[.exe]`
   - If found: `mkdir -p "$BIN_DIR"`, move to new location, `rm -f` old binary
   - Print: "Migrated binary to ${BIN_DIR}/"

5. SHA256 verification function `verify_checksum()`:
   - Accept params: `$file` and `$expected_hash`
   - Try `sha256sum "$file" | awk '{print $1}'` first
   - Fallback: `shasum -a 256 "$file" | awk '{print $1}'`
   - If neither tool available: `echo "Warning: No SHA256 tool found, skipping verification" >&2; return 0`
   - Compare hashes. On mismatch: print error with expected vs actual, `return 1`

6. Download function `download_binary()`:
   - Construct archive name: `${PROJECT}_${VERSION#v}_${PLATFORM_OS}_${PLATFORM_ARCH}.tar.gz` (or `.zip` for Windows)
   - Construct URL: `https://github.com/${REPO}/releases/download/${VERSION}/${archive_name}`
   - Construct checksums URL: `https://github.com/${REPO}/releases/download/${VERSION}/${PROJECT}_${VERSION#v}_checksums.txt`
   - Create temp directory: `tmp_dir=$(mktemp -d)`
   - Download archive: `curl -fsSL -o "${tmp_dir}/${archive_name}" "$url"` -- if fails, `rm -rf "$tmp_dir"; return 1`
   - Size check: file must be > 1000 bytes (catches HTML error pages)
   - Download checksums: `curl -fsSL -o "${tmp_dir}/checksums.txt" "$checksums_url"` -- if fails, print warning, skip verification
   - Extract expected hash: `grep "$archive_name" "${tmp_dir}/checksums.txt" | awk '{print $1}'`
   - Call `verify_checksum "${tmp_dir}/${archive_name}" "$expected_hash"` -- if fails, `rm -rf "$tmp_dir"; return 1`
   - Extract binary from archive:
     - tar.gz: `tar -xzf "${tmp_dir}/${archive_name}" -C "$tmp_dir" cc-discord-presence`
     - zip (Windows): `unzip -q "${tmp_dir}/${archive_name}" cc-discord-presence.exe -d "$tmp_dir" 2>/dev/null`
   - Move extracted binary: `mv "${tmp_dir}/${extracted}" "$BINARY"`
   - `chmod +x "$BINARY"` (skip on Windows)
   - Cleanup: `rm -rf "$tmp_dir"`
   - Print: "Downloaded and verified cc-discord-presence ${VERSION}"

7. Build fallback function `build_from_source()` (keep existing logic, updated paths):
   - `find_source_dir()` checks `${CLAUDE_PLUGIN_ROOT:-$HOME/.claude/plugins/marketplaces/dsrcode}` then `$HOME/Projects/cc-discord-presence`
   - `command -v go` check
   - Build with: `go build -ldflags "-X main.version=${VERSION#v}" -o "$BINARY" .`
   - `chmod +x "$BINARY"` (skip on Windows)

8. Install help function `show_install_help()` (English per user decision):
   - Print: "cc-discord-presence: Installation Failed"
   - Option 1: "Download manually from https://github.com/${REPO}/releases/tag/${VERSION}"
   - Option 2: "Install Go (https://go.dev/dl/) and restart Claude Code"
   - Print binary location: "Place binary in: ${BIN_DIR}/"

9. `ensure_binary()`: Try `download_binary`, then `build_from_source`, then `show_install_help; return 1`

10. Version check and binary acquisition (NO `set -e` until after this section per DIST-04):
    - Run `migrate_old_binary` first
    - `mkdir -p "$BIN_DIR"`
    - If binary missing: call `ensure_binary`
    - If binary exists: check version via `"$BINARY" --version 2>/dev/null | awk '{print $NF}'`
      - Normalize: strip `v` prefix from both current and expected
      - Special case: if current is "dev" or "unknown", skip update (local dev build)
      - If mismatch: `kill_daemon_if_running`, `rm -f "$BINARY"`, `ensure_binary`

11. `set -e` starts HERE (after binary secured)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Releases -> local binary | Downloaded binary crosses network trust boundary |
| checksums.txt -> verification | Checksum file used to validate binary integrity |
| CLAUDE_PLUGIN_DATA -> binary storage | Plugin data directory used for persistent binary storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering | Downloaded binary | mitigate | SHA256 checksum verification after download (verify_checksum function); checksums.txt downloaded from same HTTPS origin; if verification fails, binary is deleted and not executed |
| T-05-06 | Tampering | Partial download | mitigate | Atomic download via temp file + mv; size check > 1000 bytes catches HTML error pages; binary never replaces existing until fully verified |
| T-05-07 | Denial of Service | Network unavailable | mitigate | Graceful fallback: download fails -> try go build -> show install help; no crash, no silent failure |
| T-05-08 | Information Disclosure | Download URL | accept | GitHub repo is public; download URL contains repo name and version only -- no secrets |
| T-05-09 | Spoofing | checksums.txt origin | accept | Both binary and checksums downloaded from same GitHub HTTPS origin; MITM would need to compromise TLS for both; sufficient for ASVS L1 |
| T-05-10 | Elevation of Privilege | Binary execution | mitigate | Binary is chmod +x only after checksum verification passes; Windows .exe does not need chmod; binary runs as same user (no privilege escalation) |
</threat_model>
